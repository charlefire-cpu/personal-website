# Personal Website Documentation and Specifications

## Project Name
**Charmaine Buffe Personal Website**

## Document Version
**Version:** 1.0  
**Website Type:** Static Personal Portfolio Website  
**Primary User:** Charmaine Buffe  
**Target Release:** Initial static website version

---

## 1. Project Overview

The project is a simple static personal website for **Charmaine Buffe**, a **Scrum Master and Project Manager** with more than 10 years of IT experience across software engineering, Agile delivery, enterprise releases, SDLC, stakeholder management, and banking/financial systems projects.

The website will act as an online professional portfolio that summarizes Charmaine's background, competencies, work experience, education, and contact options. It is intended to be lightweight, responsive, easy to maintain, and deployable using static hosting platforms.

---

## 2. Purpose of the Website

The website aims to:

- Present Charmaine's professional profile in a clean and accessible format.
- Highlight her experience as a Scrum Master, Project Manager, and former Software Engineer.
- Showcase her Agile, project management, technical, and stakeholder management capabilities.
- Provide recruiters, hiring managers, clients, and professional contacts with a quick overview of her background.
- Offer a simple way to view or download her resume and contact her professionally.

---

## 3. Target Audience

The website is intended for:

- Recruiters
- Hiring managers
- Potential employers
- Project stakeholders
- Professional contacts
- Clients looking for Scrum Master, Project Manager, or IT delivery experience
- Technology, banking, and financial services organizations

---

## 4. Website Scope

### 4.1 In Scope

The initial static website will include:

- Home / Hero section
- About section
- Core competencies section
- Technical skills section
- Professional experience section
- Education section
- Resume download section

- Analytics dashboard

## 11. Accessibility Requirements

The website should follow basic accessibility standards:

- Use semantic HTML elements such as `header`, `main`, `section`, and `footer`.
- Use proper heading hierarchy from `h1` to `h3`.
- Ensure text has enough contrast against the background.
- Provide visible focus states for links and buttons.
- Add descriptive `aria-label` values where needed.
- Avoid relying on color alone to communicate meaning.
- Ensure navigation can be used with a keyboard.

## 15. Content Management Recommendation

Since this is a static website, content can be stored in a single data file for easier updates.

Example:

```text
src/data/profile.ts
```

This file may contain:

- Name
- Professional title
- Summary
- Skills
- Work experience
- Education
- Resume path
- Contact links

This approach makes the website easier to maintain because content updates can be done in one file without editing multiple components.
